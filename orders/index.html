Orders

The flow for processing an order is to first create an order which if successful will return an OrderId, then to pay for the order referncing the order by Id.

Note

All API calls require authentication with bearer tokens via the Authorization header. See Authentication

Paying for an Orders

In order to pay for an order a token must be provided which has the payment scope.


curl -X POST --header 'Accept: application/json'  --header 'Authorization: Bearer {TOKEN_HERE}' 'https://www.parcel2go.com/api/orders/123/paywithprepay'
